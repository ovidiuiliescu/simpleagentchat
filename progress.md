# simpleagentchat implementation progress

## Goal

Implement `specs.md` fully while following `mo.md`: work in meaningful slices, validate through public command-line and browser/server workflows, and commit after each slice.

## Slice plan

- [x] Slice 1: Core CLI foundation: Git-root discovery, initialization, safe name/path helpers, role files, gitignore/doc blocks, message storage, `say`, `fetch`, generated `chat.html` and `ui.html`.
- [x] Slice 2: Goal workflows: status files, `goal status`, `goal done`, `goal undone`, `goal recheck`, status reset semantics.
- [x] Slice 3: Server/UI/assets: `serve`, loopback HTTP endpoints, browser UI, role/goal/memory changes, human messages, asset upload/serving.
- [x] Slice 4: Acceptance hardening: temp-repo CLI scenarios, negative no-Git scenario, server/API checks, browser validation, security/path probes.
- [x] Slice 5: Live server chat rendering: dedicated role/goal add and rename controls, asset delete, SSE notifications, API-rendered Markdown chat, robust scroll preservation, and explicit `export-html`.
- [x] Slice 6: Analyzer portability: keep the single-file app clean under stricter host repository analyzers that treat latest-recommended warnings as errors.
- [x] Slice 7: Agent listening guidance: make `HOW_TO_CHAT.md` explicit that joined agents keep listening with long waits until the goal is done or they are explicitly told not to listen.
- [x] Slice 8: Goal page status visibility: show current per-role status for every goal, refresh it live from goal status events, improve required-field validation, and widen the management UI.
- [x] Slice 9: Windows file-based concurrency guidance: document `dotnet run --file ... --no-build -- ...` for agent commands that run beside `serve` or long-poll fetches.
